## 2026-07-09 — Identity en español + vistas depuradas (Bloque 2)

Pulido de las páginas de Identity, que hasta ahora salían en inglés (Default UI) con opciones que no aplican.
**Método B** (acordado con el usuario): overrides a mano, sin instalar el scaffolder, copiando la estructura
de la Default UI 10.0.9 (fuente traída del repo de ASP.NET Core vía PowerShell — curl del sandbox no sale a
GitHub por la interceptación TLS de Norton).

Decisión de arquitectura clave: no se replica el patrón interno `abstract Model` + `internal Model<TUser>` +
`[IdentityDefaultUI]` (API no soportada); cada override usa un **PageModel concreto público tipado a
`IdentityUser`** con inyección normal. El override gana a la Default UI por **ruta física** en
`Areas/Identity/Pages/...`, sin tocar `Program.cs`.

Páginas creadas: Login (ES, **sin** registro ni proveedores externos), ForgotPassword/ResetPassword +
confirmaciones, Logout, Manage/Index (Perfil), Manage/ChangePassword, `_ManageNav` (menú reducido a
Perfil/Contraseña/2FA), `_Layout` de Manage, y `_ViewImports` del área (necesario para resolver los PageModel).
**2FA se mantiene en inglés** (servido por Identity desde su ensamblado; decisión del usuario: es un extra).
El middleware que bloquea el registro público no se toca (Register sigue devolviendo 404 a anónimos).

`ForgotPassword` conserva el gate de "email confirmado" del original; como el admin sembrado ya nace con
`EmailConfirmed=true`, la recuperación le funciona. Nota para el futuro alta de admins: crearlos también con
`EmailConfirmed=true`. Verificado por HTTP (login 302, textos ES, Register 404, menú sin
Email/PersonalData/External, CSP, 0 inline). 125 tests verdes. Commit `c5ded0a`.
